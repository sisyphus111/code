00 从零构建 C++ 项目

这一节解决的不是某道算法题，而是更基础的问题：拿到一台机器和一个空目录后，怎样创建源码、编译、运行、组织多文件代码，并用 CMake 管理项目。

先建立整体模型：你写的是源码；编译器把源码变成可执行文件；终端负责启动编译器和程序；CMake 负责描述并生成整套构建规则。

1. 命令行：在终端里操作项目

终端中的每条命令都在一个“当前目录”下执行。先学会确认位置、查看文件、创建目录和切换目录。

命令作用示例
pwd显示当前目录的完整路径pwd
ls列出当前目录内容ls -la
cd切换目录；.. 表示上一级cd project、cd ..
mkdir创建目录mkdir -p src include
touch创建空文件touch src/main.cpp
cp复制文件或目录cp input.txt input.bak
mv移动或重命名mv old.cpp main.cpp
rm删除文件；通常无法从回收站恢复rm old.cpp
路径要分清：main.cpp 和 ./main.cpp 都指当前目录中的文件；../main.cpp 指上一级目录中的文件；以 / 开头的是绝对路径。

从空目录创建项目：

mkdir hello_cpp
cd hello_cpp
mkdir src include
touch src/main.cpp
ls -R

2. C++ 程序从源码到运行

最小程序只需要一个入口函数 main：

#include <iostream>

int main() {
    std::cout << "Hello, C++!\n";
    return 0;
}

用编译器直接构建。macOS 常见 clang++，Linux 和很多机试环境常见 g++；二者在这里的用法基本一致。

g++ -std=c++17 -Wall -Wextra -O2 src/main.cpp -o hello
./hello
参数含义
-std=c++17使用 C++17 标准；机试前应确认允许的标准版本
-Wall -Wextra打开常用警告，尽早发现可疑代码
-O2开启常用优化；调试阶段也可以先用 -O0 -g
-o hello把生成的可执行文件命名为 hello
./hello运行当前目录下的可执行文件；不能只写 hello

标准输入、标准输出与重定向

算法题通常从标准输入读取，向标准输出写结果：

#include <iostream>

int main() {
    int a, b;
    std::cin >> a >> b;
    std::cout << a + b << '\n';
}
echo "3 5" | ./hello          # 把字符串作为程序输入
./hello < input.txt           # 从文件读取输入
./hello < input.txt > out.txt # 输入和输出都重定向
diff out.txt answer.txt       # 比较实际输出与标准答案

3. 编译到底经历了什么

  1. 预处理：展开 #include、宏和条件编译。
  2. 编译：检查 C++ 语法与类型，把每个源文件变成目标文件。
  3. 链接：把目标文件和所需库连接成最终可执行文件。

因此错误也分两类：看到 syntax error、类型不匹配通常是编译错误；看到 undefined reference 或 Undefined symbols 通常是链接阶段找不到某个函数的实现。

4. 从单文件变成多文件项目

常见结构是头文件声明接口，源文件提供实现，main.cpp 负责组织程序流程：

hello_cpp/
├── CMakeLists.txt
├── include/
│   └── calculator.h
└── src/
    ├── calculator.cpp
    └── main.cpp

include/calculator.h：

#pragma once

int add(int a, int b);

src/calculator.cpp：

#include "calculator.h"

int add(int a, int b) {
    return a + b;
}

src/main.cpp：

#include <iostream>
#include "calculator.h"

int main() {
    std::cout << add(3, 5) << '\n';
}

暂时不用 CMake 时，可以把两个源文件同时交给编译器，并用 -I 指定头文件目录：

g++ -std=c++17 -Wall -Wextra -Iinclude \
    src/main.cpp src/calculator.cpp -o calculator
./calculator

5. CMake：描述项目怎样构建

CMake 不是编译器。它读取 CMakeLists.txt，生成适合当前平台的构建系统，再由该构建系统调用编译器。

项目根目录的 CMakeLists.txt：

cmake_minimum_required(VERSION 3.16)
project(hello_cpp LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

add_executable(calculator
    src/main.cpp
    src/calculator.cpp
)

target_include_directories(calculator PRIVATE include)
target_compile_options(calculator PRIVATE -Wall -Wextra)
CMake 命令作用
project(...)声明项目名称及使用的语言
add_executable(...)创建一个可执行目标，并列出组成它的源文件
target_include_directories(...)为指定目标添加头文件搜索目录
target_compile_options(...)为指定目标添加编译选项

标准的三步工作流

# 1. 配置：读取当前目录，生成构建规则到 build/
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug

# 2. 构建：让 CMake 调用底层构建工具和编译器
cmake --build build

# 3. 运行
./build/calculator

-S . 指源码根目录，-B build 指构建产物目录。把产物放进独立的 build/ 叫作 out-of-source build，可以避免目标文件和临时文件污染源码目录。

修改代码后需要执行哪一步

  • 只修改了 .cpp 或 .h：通常重新执行 cmake --build build。
  • 修改了 CMakeLists.txt：构建时通常会自动重新配置；不确定时再运行一次配置命令。
  • 切换编译器、生成器或重要配置：删除旧 build/ 后重新配置，避免缓存冲突。

6. Debug 与 Release

模式特点适合场景
Debug包含调试信息、优化较少，便于定位代码开发、断点调试、排错
Release开启优化，运行通常更快，但调试体验较差性能测试、最终提交前验证
 
cmake -S . -B build-debug -DCMAKE_BUILD_TYPE=Debug
cmake --build build-debug

cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release
cmake --build build-release

7. 最常见的错误怎么判断

现象先检查什么
command not found: cmakeCMake 是否安装，以及可执行文件是否位于 PATH
fatal error: xxx.h: No such file头文件名是否正确，include 目录是否通过 -I 或 CMake 加入
undefined reference函数是否只有声明没有实现，或实现所在的 .cpp 是否加入构建目标
permission denied是否在运行源码文件，或可执行文件是否缺少执行权限
./program: no such file编译是否成功、程序实际生成在哪个目录、文件名是否正确
修改后运行结果没变化是否重新构建，以及运行的是不是另一个目录中的旧程序
排错原则：先看终端输出的第一条错误，而不是最后一条；后面的错误经常只是第一处失败引发的连锁反应。

8. 从零开始时的固定动作

  1. 运行 pwd 和 ls，确认自己在哪里、已有些什么。
  2. 建立 src/、include/ 和根目录的 CMakeLists.txt。
  3. 先写一个能打印固定字符串的最小 main.cpp，打通构建链路。
  4. 用 cmake -S . -B build 配置，用 cmake --build build 构建。
  5. 运行最小程序，确认输入、输出和工作目录符合预期。
  6. 再逐步添加业务模块或算法；每增加一小步就重新构建验证。
  7. 提交前用多组输入测试，并用 Release 模式检查性能。

9. 一分钟速查

# 看环境
pwd
ls -la
g++ --version
cmake --version

# 直接编译单文件
g++ -std=c++17 -Wall -Wextra -O2 main.cpp -o main
./main < input.txt

# 构建 CMake 项目
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
./build/程序名

掌握这一节的标准：不依赖 IDE 的“运行”按钮，也能在空目录中创建并运行一个多文件 C++ 项目；看到编译或链接错误时，知道应该从哪一层开始检查。
