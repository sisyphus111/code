# 机考代码题题型与考点总结

这份材料解决三个问题：

- 题目一看，应该往哪个题型归类。
- 数据范围一看，复杂度大概应该是多少。
- 考场上先拿什么分，哪里最容易错。

## 读题三步

### 1. 先看数据范围估复杂度

| 数据规模 | 常见可接受复杂度 | 典型方向 |
|---|---:|---|
| `n <= 20` | `O(2^n)` / 状压 / 回溯 | 子集、排列、状态压缩 |
| `n <= 100` | `O(n^3)` | Floyd、区间 DP、小规模枚举 |
| `n <= 1000` | `O(n^2)` | 普通 DP、枚举两端点 |
| `n <= 1e5` | `O(n log n)` / `O(n)` | 排序、堆、二分、图遍历 |
| `n <= 1e6` | `O(n)` | 线性扫描、前缀和、桶 |
| `值域 <= 1e9` | `O(log V)` / `O(sqrt V)` | 二分答案、数论 |

注意：多组测试时总规模更重要。题面如果写“所有测试点的 `sum n <= 1e5`”，就按总规模设计。

### 2. 再看关键词识别题型

| 题面信号 | 优先想到 |
|---|---|
| “连续子数组/区间和/多次询问区间” | 前缀和、差分、滑动窗口 |
| “最小化最大值/最大化最小值/是否可行” | 二分答案 |
| “最短步数/迷宫/变换一次算一步” | BFS |
| “连通块/岛屿/可达性” | DFS/BFS、并查集 |
| “最短路/边权非负” | Dijkstra |
| “依赖关系/先修课程/任务顺序” | 拓扑排序 |
| “选或不选/容量限制” | 背包 DP |
| “最长/最少/方案数，且有明显状态转移” | DP |
| “第 k 大/动态维护最大最小” | 堆、`priority_queue` |
| “括号/最近更大更小/柱状图” | 栈、单调栈 |
| “频次/去重/是否出现过/两数之和” | 哈希表 |
| “排序后相邻/区间合并/贪心选择” | 排序、贪心 |

### 3. 最后决定拿分策略

- 先写能 20-30 分钟稳定 AC 的题。
- 中等题先在纸上写状态、边界和复杂度，再动代码。
- 难题先想暴力和部分分：`n <= 1000` 的子任务可能能用 `O(n^2)`。
- 每道题提交前至少造一个边界样例：空、单元素、全相同、最大值、无解。

## 题型地图

### 1. 模拟与枚举

常见场景：日期、进制、字符串格式、矩阵旋转、排行榜、游戏过程。

考点：

- 读题精确，状态变量清楚。
- 下标和边界。
- 多条件排序。
- 枚举范围别漏。

常用工具：

- `sort`
- `stable_sort`
- `vector`
- `map/unordered_map`
- 自定义结构体和比较函数

易错点：

- 输出格式多空格或少换行。
- `int` 溢出。
- 多组数据没有清空状态。
- 题目要求稳定排序却用了普通排序后未处理 tie-break。

### 2. 字符串处理

常见场景：单词统计、回文、模式匹配、括号、路径解析、表达式。

考点：

- `string` 增删改查。
- `getline` 读取整行。
- `stringstream` 按空白切词。
- 字符分类：`isdigit`、`isalpha`、`tolower`。

常见算法：

- 双指针判断回文。
- 哈希表统计频率。
- KMP 用于严格模式匹配。
- 栈用于括号和表达式。

易错点：

- `cin >> x` 后直接 `getline` 会读到残留换行，需要先 `cin.ignore()`。
- `s.size()` 是无符号类型，倒序循环别写 `for (int i = s.size() - 1; i >= 0; --i)` 处理空串。

### 3. 哈希、映射、集合

常见场景：出现次数、去重、编号映射、两数之和、分组。

选型：

- `unordered_map`：平均 `O(1)`，适合查找计数。
- `map`：`O(log n)`，按 key 有序。
- `unordered_set`：平均 `O(1)` 去重。
- `set`：有序集合、找前驱后继。

易错点：

- `mp[x]` 会在 key 不存在时插入默认值；只判断存在性用 `count` 或 `find`。
- 自定义 key 用 `unordered_map` 需要自定义 hash。
- 数据被卡哈希时 `unordered_map` 可能退化，普通机试一般不刻意卡，但要知道风险。

### 4. 双指针与滑动窗口

常见场景：有序数组两数和、最长无重复子串、满足条件的最短区间。

模式：

- 对撞指针：`l=0, r=n-1`，用于有序数组。
- 同向双指针：`r` 扩张，`l` 收缩，用于连续区间。

判断标准：

- 区间满足条件后，移动左端可以保持或恢复条件。
- 题目要求连续子数组/子串。

易错点：

- 窗口更新顺序：先加入右端，再 while 收缩，再更新答案，还是先更新答案，要和题意一致。
- 计数型窗口要维护“条件是否满足”，不要每次重新扫窗口。

### 5. 前缀和与差分

常见场景：区间和、多次区间查询、矩阵子区域、区间批量加。

核心公式：

- 一维前缀：`sum[l..r] = pre[r + 1] - pre[l]`
- 二维前缀：`S(x1,y1,x2,y2) = pre[x2+1][y2+1] - pre[x1][y2+1] - pre[x2+1][y1] + pre[x1][y1]`
- 差分区间加：`diff[l] += v, diff[r + 1] -= v`

易错点：

- 推荐统一用 `pre` 长度 `n + 1`，减少 `l=0` 特判。
- 累加和通常用 `long long`。

### 6. 二分与二分答案

常见场景：有序查找、最小可行值、最大可行值。

普通二分：

```cpp
int l = 0, r = n; // [l, r)
while (l < r) {
    int mid = l + (r - l) / 2;
    if (a[mid] >= x) r = mid;
    else l = mid + 1;
}
```

二分答案：

- 先写 `bool check(ans)`。
- `check(x)` 必须具有单调性。
- 明确找“最小可行”还是“最大可行”。

易错点：

- `mid = (l + r) / 2` 可能溢出。
- `while (l <= r)` 和 `while (l < r)` 不要混用模板。
- 浮点二分用固定次数循环更稳。

### 7. 栈、队列、单调结构

栈：

- 括号匹配。
- 表达式求值。
- 递归过程模拟。

单调栈：

- 下一个更大/更小元素。
- 柱状图最大矩形。
- 每个元素作为最小值/最大值的贡献。

队列：

- BFS。
- 滑动窗口最大值用单调队列。

易错点：

- `stack`、`queue` 不能遍历，只能看 `top/front/back`。
- 单调栈里“相等元素”是弹还是不弹，取决于是否要避免重复计数。

### 8. 堆与贪心

常见场景：第 k 大、合并石子、任务调度、实时维护最大/最小。

堆：

- 默认 `priority_queue<int>` 是大根堆。
- 小根堆用 `priority_queue<int, vector<int>, greater<int>>`。

贪心：

- 先排序，再局部最优选择。
- 常见证明方式：交换论证、反证、维护不变量。

易错点：

- 贪心题不要只靠直觉，至少能解释“为什么当前选择不会让未来更差”。
- 堆里存 pair 时默认按字典序比较。

### 9. DFS、回溯与剪枝

常见场景：全排列、组合、子集、搜索路径、数独、小规模选择。

骨架：

- 递归参数表示当前层状态。
- 到达终止条件时记录答案。
- 做选择、递归、撤销选择。

剪枝：

- 当前答案已经不可能更优。
- 剩余元素不足。
- 排序后跳过重复分支。

易错点：

- 引用传递的路径要回溯恢复。
- 全局答案在多组测试中要清空。
- 递归深度大时注意爆栈。

### 10. BFS 与网格题

常见场景：迷宫最短路、腐烂橘子、岛屿数量、状态一步一步变化。

适用：

- 边权相同，求最短步数，用 BFS。
- 只求连通性，DFS/BFS 都行。

常用方向数组：

```cpp
int dx[4] = {1, -1, 0, 0};
int dy[4] = {0, 0, 1, -1};
```

易错点：

- BFS 最好入队时就标记 visited，避免重复入队。
- 网格边界：`0 <= nx && nx < n && 0 <= ny && ny < m`。
- 多源 BFS：先把所有起点入队。

### 11. 图论基础

常见场景：社交网络、路径、依赖、连通关系。

必会：

- 邻接表建图。
- DFS/BFS 遍历。
- 并查集判断连通。
- 拓扑排序处理 DAG。
- Dijkstra 处理非负边权最短路。

选型：

- 无权最短路：BFS。
- 非负权：Dijkstra。
- 有负权但无负环：Bellman-Ford/SPFA，夏令营机试较少。
- 任意两点最短路且 `n <= 500`：Floyd。

易错点：

- 无向图要加两条边。
- Dijkstra 里弹出的旧状态要跳过：`if (d != dist[u]) continue;`
- 拓扑排序结束后，如果输出数量小于 `n`，说明有环。

### 12. 动态规划

常见场景：最优值、方案数、可行性，且问题能拆成子问题。

思考顺序：

1. 状态：`dp[i]` / `dp[i][j]` 表示什么。
2. 转移：从哪些旧状态转来。
3. 初始化：最小规模答案。
4. 遍历顺序：旧状态必须已经计算。
5. 答案位置：是 `dp[n]`、最大值，还是多个状态求和。

高频类型：

- 线性 DP：爬楼梯、最大子数组、打家劫舍。
- 背包：01 背包、完全背包。
- LIS：最长上升子序列。
- 区间 DP：合并石子、括号匹配。
- 树形 DP：树上选择、最大独立集。

易错点：

- 方案数要取模。
- 不可能状态要初始化为极小/极大。
- 01 背包容量倒序，完全背包容量正序。

### 13. 数学与位运算

常见场景：最大公约数、质数、组合计数、快速幂、异或性质。

必会：

- `gcd`
- 快速幂取模
- 质数判断到 `sqrt(n)`
- 埃氏筛
- 组合数小规模递推
- 位运算枚举子集

易错点：

- 取模减法要加 `MOD`：`(a - b + MOD) % MOD`。
- 乘法可能溢出，用 `long long`。
- `1 << n` 在 `n >= 31` 时有风险，用 `1LL << n`。

### 14. 树与二叉树

常见场景：层序遍历、前中后序、最近公共祖先、树形 DP。

LeetCode 模式常给 `TreeNode*`，ACM 模式通常要自己建邻接表。

必会：

- 递归遍历。
- BFS 层序。
- 父节点数组。
- 树上 DFS 统计子树。

易错点：

- 树是无向边输入时 DFS 要传 `parent`，避免走回去。
- 二叉树空节点要判空。

## 考场优先级

如果时间有限，按这个顺序保底：

1. 模拟、排序、字符串、哈希。
2. 二分、双指针、前缀和。
3. DFS/BFS、并查集、拓扑。
4. Dijkstra、基础 DP。
5. 更复杂的 DP、数论、数据结构。

## 提交前检查清单

- 数组大小是否比最大下标多 5-10。
- 累加、乘法、距离是否用 `long long`。
- 多组输入是否清空 `vector`、`queue`、`visited`、`dist`。
- 下标统一 0-based 还是 1-based。
- 图是有向还是无向。
- 二分边界是否会死循环。
- BFS 是否入队时标记。
- 输出是否严格符合题面。
- 本地调试输出是否删掉或改成 `cerr`。

## 两个月学习深度要求

前面的内容更像地图。这一节把每类题拆成“学到什么程度才算过关”。

### 模拟题

学习目标：

- 能把复杂题意拆成若干状态变量。
- 能处理排序、多条件判断、计数、字符串解析、日期时间。
- 能在代码里保持清晰的状态更新顺序。

训练方法：

- 每题先写变量表：每个变量代表什么，什么时候更新。
- 对长模拟题，先写伪代码，再写 C++。
- 每做完一道题，手动跑一遍样例过程。

建议练习主题：

- 日期合法性、闰年、星期计算。
- 成绩排序、排行榜、奖牌榜。
- 字符串格式化、路径规范化。
- 棋盘/矩阵操作。
- 简单游戏过程模拟。

完成标准：

- 200 行以内模拟题能保持变量不乱。
- 输出格式和边界基本不出错。
- 能自己造样例验证关键分支。

### 排序与比较函数

学习目标：

- 熟悉升序、降序、多关键字排序。
- 理解 comparator 必须满足严格弱序。
- 会用 `stable_sort` 处理稳定性需求。

训练方法：

- 对每道排序题写出比较规则的优先级。
- 手动构造 tie case，检查 comparator。

常见比较规则：

```cpp
sort(a.begin(), a.end(), [](const Node& x, const Node& y) {
    if (x.score != y.score) return x.score > y.score;
    if (x.age != y.age) return x.age < y.age;
    return x.name < y.name;
});
```

完成标准：

- 看到“若相同则按...”能自然写出多关键字 comparator。
- 不写 `return a <= b` 这种非法比较。

### 哈希与计数

学习目标：

- 熟练用 `unordered_map` 做频次统计。
- 熟练用 `map` 做有序统计。
- 掌握“编号映射”：把字符串、人名、坐标映射到整数 id。

训练方法：

- 每次使用 `mp[x]` 前问自己：我是否真的想插入默认值？
- 对大输入练 `reserve` 和 `max_load_factor`。

典型模式：

```cpp
unordered_map<string, int> id;
int get_id(const string& s) {
    if (!id.count(s)) id[s] = id.size();
    return id[s];
}
```

完成标准：

- 能用哈希把 `O(n^2)` 查找降到 `O(n)` 或 `O(n log n)`。
- 能处理 key 为字符串、整数、pair 的常见情况。

### 前缀和、差分、区间

学习目标：

- 看到“多次区间查询”优先想到前缀和。
- 看到“多次区间修改，最后查询结果”优先想到差分。
- 会处理二维矩阵的子矩形和。

训练方法：

- 全部统一用 `pre[n + 1]`。
- 写公式前先画出 `[l, r]` 和 `[0, i)` 的关系。

进阶主题：

- 前缀和 + 哈希：统计和为 `k` 的子数组数量。
- 前缀和 + 取模：最长/数量问题。
- 二维前缀和：矩阵区域查询。

完成标准：

- 一维和二维公式能默写。
- 能解释为什么差分还原时要做一次前缀累加。

### 二分答案

学习目标：

- 能把优化问题转成判定问题。
- 能证明 `check(x)` 单调。
- 会区分找最小可行和最大可行。

训练方法：

- 每道二分答案题必须写一句：`x` 越大，条件越容易/越难满足。
- 先暴力写 `check`，再优化 `check`。

典型题意：

- 最小化最大负载。
- 最大化最小间距。
- 在限定时间内能否完成。
- 切木头、分组、运输、分配。

完成标准：

- 不混用模板。
- 浮点二分和整数二分分得清。
- 能把答案范围设对。

### 搜索专题

学习目标：

- DFS 负责枚举和连通性。
- BFS 负责等权最短路。
- 回溯负责构造所有方案。
- 剪枝负责让指数搜索尽量少走。

训练方法：

- DFS 题写清楚递归参数。
- BFS 题写清楚状态和一步转移。
- 回溯题写清楚选择列表和撤销操作。

搜索状态设计：

```text
状态 = 当前位置 + 已经使用的信息 + 当前代价/步数
转移 = 一步能做什么
终止 = 找到目标 / 枚举完成 / 不可能继续
```

完成标准：

- 迷宫最短路能一次写对。
- 全排列/组合/子集能默写。
- 能判断什么时候需要 visited，什么时候需要回溯恢复。

### 图论专题

学习目标：

- 熟练建邻接表。
- 会用 BFS/DFS 处理连通和可达。
- 会用拓扑排序处理依赖。
- 会用 Dijkstra 处理非负权最短路。
- 会用并查集处理动态合并和连通性。

训练方法：

- 建图前标注：有向/无向、带权/无权、编号起点。
- 最短路题先看边权：无权、非负、有负权。
- 依赖题先想是否 DAG。

完成标准：

- 能独立写 1-based 到 0-based 转换。
- 能解释 BFS 和 Dijkstra 的区别。
- 能检测拓扑排序中的环。

### DP 专题

学习目标：

- 用“状态、转移、初始化、遍历顺序、答案”五步法。
- 掌握线性 DP、背包、LIS、路径 DP、区间 DP 入门。
- 能从递归/暴力枚举推导出 DP。

训练方法：

每道 DP 题必须写：

```text
dp 状态：
转移：
初始化：
遍历顺序：
答案：
复杂度：
```

典型状态：

- `dp[i]`：前 `i` 个元素的最优值。
- `dp[i][j]`：前 `i` 个元素，在容量/状态 `j` 下的最优值。
- `dp[l][r]`：区间 `[l, r]` 的最优值。
- `dp[u][state]`：树上以 `u` 为根，处于某状态的最优值。

完成标准：

- 能把 01 背包和完全背包的循环方向说清楚。
- 能判断 `dp` 初始化为 0、`INF` 还是 `-INF`。
- 能处理方案数取模。

### 数学专题

学习目标：

- 掌握最大公约数、快速幂、筛法、组合数基础。
- 熟悉取模规则。
- 会用位运算表示集合。

训练方法：

- 对每个公式造小样例验证。
- 涉及乘法和幂时默认检查溢出。

完成标准：

- 快速幂能默写。
- 子集枚举能写对。
- 能解释异或的基本性质：自反、交换、结合。

## 题型混合时的判断顺序

真实机试不会告诉你标签。可以按这个顺序问：

1. 是否只是按题意实现？如果是，模拟。
2. 是否有连续区间？前缀和、双指针、滑动窗口。
3. 是否有“最小化最大/最大化最小”？二分答案。
4. 是否有网格、一步一步走、最短步数？BFS。
5. 是否有连通关系？DFS/BFS/并查集。
6. 是否有依赖顺序？拓扑排序。
7. 是否有边权最短路？Dijkstra/Floyd。
8. 是否是选或不选、最优值、方案数？DP。
9. 是否要动态维护最大/最小？堆。
10. 是否要最近更大/更小？单调栈。

## 专题练习配比

两个月里不建议平均分配时间。建议按收益分配：

| 专题 | 时间占比 | 原因 |
|---|---:|---|
| 模拟、字符串、排序、哈希 | 20% | 机试保分核心 |
| 前缀和、二分、双指针 | 15% | 中等题高频 |
| 搜索、并查集、图遍历 | 20% | 网格/图题常见 |
| 图论最短路、拓扑 | 15% | 学校机试常用筛选题 |
| DP | 20% | 拉开差距 |
| 数学、位运算、杂项 | 10% | 补齐边界能力 |

## 从暴力到优化的训练

很多中等题不是一眼想出最优解，而是从暴力改出来：

```text
O(n^3) 枚举区间求和
-> 用前缀和把求和变 O(1)
-> O(n^2)

O(n^2) 查两数和
-> 用哈希记录已见元素
-> O(n)

O(n^2) 找每个元素右侧更大值
-> 用单调栈
-> O(n)

O(n * answer_range) 枚举答案
-> 对答案二分
-> O(n log V)
```

训练时可以先写暴力理解题意，再问自己“重复计算在哪里”。
