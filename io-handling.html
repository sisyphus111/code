基础 02：C++ 输入输出与 CLI 工具

会构建项目以后，下一步是让程序真正与外界交互：接收标准输入或命令行参数、读取文件、输出结果、报告错误，并用退出码告诉调用者执行是否成功。

整体模型：一个 CLI 程序有三类输入——命令行参数、标准输入、文件；有三类输出——标准输出、标准错误、文件；最后还会返回一个整数退出码。

1. 先分清参数和输入数据

./textstats --input article.txt --output result.txt
  • --input article.txt --output result.txt 是命令行参数，用于控制程序怎样运行。
  • article.txt 中的正文是输入数据，是程序真正需要处理的内容。

参数适合表达少量配置，例如输入路径、输出路径、模式和开关；大量数据应通过标准输入或文件传入。

2. 标准输入、标准输出和标准错误

C++ 流含义应该放什么
std::cin标准输入 stdin程序需要处理的数据
std::cout标准输出 stdout正常结果，便于重定向或交给下一个程序
std::cerr标准错误 stderr错误、警告和诊断信息
 
#include <iostream>
#include <string>

int main() {
    std::string name;
    if (!(std::cin >> name)) {
        std::cerr << "error: expected a name\n";
        return 1;
    }
    std::cout << "hello, " << name << '\n';
    return 0;
}

正常结果必须写入 cout，错误必须写入 cerr。这样用户执行 ./program > result.txt 时，结果进入文件，而错误仍显示在终端。

3. 用 >> 读取单词和数字

operator>> 会自动跳过空白，适合按 token 读取结构化数据：

int n;
if (!(std::cin >> n)) {
    std::cerr << "error: invalid count\n";
    return 1;
}

std::vector<int> values(n);
for (int& value : values) {
    if (!(std::cin >> value)) {
        std::cerr << "error: not enough integers\n";
        return 1;
    }
}
不要只写 std::cin >> n; 就假设成功。项目式机试里，输入可能缺失或格式错误；读取表达式本身可以转为布尔值，因此可以直接检查。

4. 空白符号与输入缓冲区的真相

机考中最容易出错的地方不是算法，而是 cin >> 和 getline 的组合以及各种隐藏的空白符。

4.1 空白符号的本质

符号名称ASCII 码出现场景
'\n'LF (Line Feed)10Unix/Linux/macOS 行尾
'\r'CR (Carriage Return)13老式 Mac 行尾
'\r\n'CRLF13 + 10Windows 行尾（两个字符）
' '空格32普通空格
'\t'Tab9制表符
关键：Windows 文本文件的每行结尾是 \r\n 两个字符，Unix/Linux 只有 \n 一个字符。如果在 Linux 上读 Windows 文件，或者提交代码到评测机，可能会遇到 \r 残留问题。

4.2 cin >> 的行为

int n;
std::string s;
std::cin >> n >> s;

cin >> 的特点：

  • 自动跳过所有空白（空格、\n、\r、\t）
  • 读到第一个非空白字符开始，读到下一个空白字符停止
  • 不会读取空白符本身，空白符留在缓冲区
// 输入：42\n（42 后面跟换行符）
int n;
std::cin >> n;  // 读取 42，\n 留在缓冲区

4.3 getline 的行为

std::string line;
std::getline(std::cin, line);

getline 的特点：

  • 不会跳过任何字符
  • 从当前位置读到 \n 停止
  • \n 被读取并丢弃，但不包含在 line 中
  • 空行会被读取为空字符串

4.4 混用的陷阱（重点！）

// 输入：42\napple banana\n
int n;
std::cin >> n;              // 读 42，\n 留在缓冲区
std::string line;
std::getline(std::cin, line); // 立即读到 \n，返回空字符串！

问题：cin >> n 读了 42，但 \n 还留在缓冲区，getline 立即读到它并返回空字符串。

解决方案 1：用 cin.ignore() 丢弃换行符

#include <limits>

int n;
std::cin >> n;
std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
std::string line;
std::getline(std::cin, line);  // 现在正常读取

解决方案 2：全部用 getline + 手动解析

std::string line;
std::getline(std::cin, line);
int n = std::stoi(line);
std::getline(std::cin, line);  // 正常读取下一行

4.5 实战场景对比

场景 1：读取若干整数

// 输入：3\n10 20 30\n
int n;
std::cin >> n;  // 读 3，留下 \n
for (int i = 0; i < n; ++i) {
    int x;
    std::cin >> x;  // cin >> 会自动跳过 \n，正常读取
}

✅ 没问题，因为 cin >> 会跳过空白。

场景 2：读整数后读字符串

// 输入：3\napple banana cherry\n
int n;
std::cin >> n;              // 读 3，留下 \n
std::string line;
std::getline(std::cin, line); // 读到 \n，返回空字符串！

❌ 有问题，必须用 cin.ignore() 或改用全部 getline。

场景 3：Windows 文件在 Linux 上读取

// Windows 文件内容：hello\r\n
std::string line;
std::getline(std::cin, line);  // line = "hello\r"，包含 \r！

❌ 有问题，\r 会留在字符串里，可能导致比较失败或输出异常。

解决：手动去除 \r

std::getline(std::cin, line);
if (!line.empty() && line.back() == '\r') {
    line.pop_back();
}

4.6 常见错误模式总结

错误代码问题修正
cin >> n; getline(cin, s);\n 残留导致读空行加 cin.ignore()
Windows 文件 getline\r 残留在字符串末尾手动 pop_back()
循环混用 >> 和 getline缓冲区状态混乱统一用一种方式
机考建议：要么全部用 cin >>（适合结构化数据），要么全部用 getline + stoi/stod（适合行输入）。混用时必须在每次 >> 后用 ignore() 清理缓冲区。

std::string line;
while (std::getline(std::cin, line)) {
    std::cout << "read: " << line << '\n';
}

6. 读取和写入文件

std::ifstream 用于读取文件，std::ofstream 用于写入文件。对象离开作用域后会自动关闭，不必手动调用 close()。

#include <fstream>
#include <iostream>
#include <string>

int main() {
    std::ifstream input("input.txt");
    if (!input) {
        std::cerr << "error: cannot open input.txt\n";
        return 1;
    }

    std::ofstream output("output.txt");
    if (!output) {
        std::cerr << "error: cannot open output.txt\n";
        return 1;
    }

    std::string line;
    while (std::getline(input, line)) {
        output << line << '\n';
    }
}
写入方式效果
std::ofstream output(path);默认覆盖已有文件
std::ofstream output(path, std::ios::app);追加到文件末尾
std::ifstream input(path, std::ios::binary);以二进制模式读取

7. 读取命令行参数：argc 与 argv

int main(int argc, char* argv[]) {
    std::cout << "argument count: " << argc << '\n';
    for (int i = 0; i < argc; ++i) {
        std::cout << "argv[" << i << "] = " << argv[i] << '\n';
    }
}

执行：

./program --input article.txt

得到的参数关系是：

argc    = 3
argv[0] = "./program"
argv[1] = "--input"
argv[2] = "article.txt"

argv[0] 是程序自身的调用名称；真正的用户参数从 argv[1] 开始。读取 --input 后必须检查后面是否还有值，不能直接访问不存在的 argv[i + 1]。

8. 手动解析常见 CLI 选项

小型机试项目不一定允许引入参数解析库，下面的写法足以处理 --help、输入文件和输出文件：

struct Options {
    std::string input_path;
    std::string output_path;
    bool help = false;
};

bool parse_options(int argc, char* argv[], Options& options) {
    for (int i = 1; i < argc; ++i) {
        std::string arg = argv[i];
        if (arg == "-h" || arg == "--help") {
            options.help = true;
        } else if (arg == "-i" || arg == "--input") {
            if (i + 1 == argc) {
                std::cerr << "error: " << arg << " needs a path\n";
                return false;
            }
            options.input_path = argv[++i];
        } else if (arg == "-o" || arg == "--output") {
            if (i + 1 == argc) {
                std::cerr << "error: " << arg << " needs a path\n";
                return false;
            }
            options.output_path = argv[++i];
        } else {
            std::cerr << "error: unknown option: " << arg << '\n';
            return false;
        }
    }
    return true;
}

9. 退出码也是程序接口

  • return 0;：执行成功。
  • 非零值：执行失败。常用 return 1; 表示一般错误，保持简单即可。
./program --input missing.txt
echo $?

调用程序、测试脚本和评测系统不会阅读你在终端上打印的中文解释来判断成功，而是检查退出码。因此“向 cerr 写清楚原因 + 返回非零退出码”应该同时完成。

10. 完整示例：文本统计 CLI

这个程序支持四种使用方式：

./textstats < article.txt
./textstats --input article.txt
./textstats --input article.txt --output result.txt
./textstats --help

未指定输入文件时读取 stdin，未指定输出文件时写入 stdout。这是 CLI 工具很实用的设计：既能独立操作文件，也能参与管道。

#include <algorithm>
#include <cctype>
#include <fstream>
#include <iostream>
#include <iterator>
#include <string>

struct Options {
    std::string input_path;
    std::string output_path;
    bool help = false;
};

bool parse_options(int argc, char* argv[], Options& options) {
    for (int i = 1; i < argc; ++i) {
        std::string arg = argv[i];
        if (arg == "-h" || arg == "--help") {
            options.help = true;
        } else if (arg == "-i" || arg == "--input" ||
                   arg == "-o" || arg == "--output") {
            if (i + 1 == argc) {
                std::cerr << "error: " << arg << " needs a path\n";
                return false;
            }
            std::string path = argv[++i];
            if (arg == "-i" || arg == "--input") {
                options.input_path = path;
            } else {
                options.output_path = path;
            }
        } else {
            std::cerr << "error: unknown option: " << arg << '\n';
            return false;
        }
    }
    return true;
}

int main(int argc, char* argv[]) {
    Options options;
    if (!parse_options(argc, argv, options)) {
        return 1;
    }
    if (options.help) {
        std::cout << "Usage: textstats [-i FILE] [-o FILE]\n";
        return 0;
    }

    std::ifstream input_file;
    std::istream* input = &std::cin;
    if (!options.input_path.empty()) {
        input_file.open(options.input_path);
        if (!input_file) {
            std::cerr << "error: cannot open input file: "
                      << options.input_path << '\n';
            return 1;
        }
        input = &input_file;
    }

    std::ofstream output_file;
    std::ostream* output = &std::cout;
    if (!options.output_path.empty()) {
        output_file.open(options.output_path);
        if (!output_file) {
            std::cerr << "error: cannot open output file: "
                      << options.output_path << '\n';
            return 1;
        }
        output = &output_file;
    }

    std::string text((std::istreambuf_iterator<char>(*input)),
                     std::istreambuf_iterator<char>());

    std::size_t lines = std::count(text.begin(), text.end(), '\n');
    if (!text.empty() && text.back() != '\n') {
        ++lines;
    }

    std::size_t words = 0;
    bool inside_word = false;
    for (unsigned char ch : text) {
        if (std::isspace(ch)) {
            inside_word = false;
        } else if (!inside_word) {
            ++words;
            inside_word = true;
        }
    }

    *output << "lines " << lines << '\n'
            << "words " << words << '\n'
            << "bytes " << text.size() << '\n';
    return 0;
}

11. 项目式机试的 I/O 检查清单

  1. 先写清楚程序接受哪些参数，以及输入数据来自 stdin 还是文件。
  2. 正常结果只写 stdout 或指定输出文件，诊断信息只写 stderr。
  3. 每次打开文件和读取关键字段后都检查状态。
  4. 参数缺值和未知参数必须报错，不能越界读取 argv。
  5. 成功返回 0，失败返回非零值。
  6. 给出简短准确的 --help，让评测者知道程序如何运行。
  7. 将核心处理逻辑与 I/O 分开，避免所有代码都堆在 main 中。
掌握标准：能写出一个同时支持 stdin/stdout 和输入输出文件的 CLI；能解析基本选项；遇到错误时能输出诊断并返回非零退出码。
